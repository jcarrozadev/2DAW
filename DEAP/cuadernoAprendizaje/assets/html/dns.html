Introducción a DNS

DNS (Domain Name System) es un sistema de nomenclatura jerárquico descentralizado para dispositivos conectados a redes IP como Internet o una red privada. Asocia información variada con nombres de dominio asignados a cada uno de los participantes.
Utiliza el puerto UDP 53.

¿Qué es un dominio?

Un dominio es un nombre que identifica a una entidad en Internet. Consiste en un conjunto de nombres que se separan por puntos y que se lee de derecha a izquierda. Por ejemplo, en el dominio www.google.com, com es el dominio de nivel superior, google es el dominio de segundo nivel y www es el subdominio. Ejemplo: www.google.com

¿Qué es un subdominio?

Un subdominio es un dominio que forma parte de un dominio de nivel superior. Por ejemplo, en el dominio www.google.com, www es el subdominio.

¿Para que nos servirá?

Su función principal es traducir (resolver) un nombre de dominio a una dirección IP.

Configurar servidor DNS

Para configurar un servidor DNS, se debe tener en cuenta que el servidor debe tener una dirección IP estática y tener instalado el servicio DNS. En Linux, se puede instalar el servicio DNS con el comando: 


sudo apt-get install bind9

Para más detalles, puedes consultar la presentación sobre cómo configurar un servidor DNS.

Luego, se debe configurar el archivo /etc/bind/named.conf.options con las opciones de configuración del servidor DNS.

Comandos Útiles de Consulta

nslookup - Permite consultar la dirección IP de un dominio.
dig - Permite consultar información de un dominio.

Tipos de registros DNS

A - Asocia un nombre de dominio a una dirección IP.
CNAME - Asocia un nombre de dominio a otro nombre de dominio.
MX - Asocia un nombre de dominio a un servidor de correo.
TXT - Asocia un nombre de dominio a un texto.
PTR - Asocia una dirección IP a un nombre de dominio.
